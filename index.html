Tasy EMR • Gestão Hospitalar

🏥 Extração de Biblioteca de Fármacos para Bombas de Infusão

1. O que é o projeto

Desenvolvimento de uma consulta SQL avançada para a extração de dados técnicos de medicamentos (doses, diluições e estabilidades) do ERP Tasy. O objetivo foi fornecer a base de dados necessária para a configuração da Biblioteca de Fármacos (Drug Library) das novas bombas de infusão da marca B Braun na unidade hospitalar HMG.

2. O Problema Técnico

  • Dados Dispersos: As informações necessárias estavam espalhadas em mais de 9 tabelas diferentes.
  • Hierarquia de Dados: Muitas doses padrão estavam cadastradas na "capa" do material, mas não na aba de "Prescrição", ou vice-versa.
  • Variações de Via: Medicamentos com vias específicas não estavam sendo vinculados corretamente.
  • Codificação Técnica: Campos retornavam siglas internas ilegíveis para a equipe assistencial.

3. A Solução Aplicada

Utilizei Joins Complexos, Lógica de Fallback (NVL e NULLIF) e Normalização com DECODE e CASE para tradução de códigos internos para texto por extenso.

4. O Impacto na Ponta

Segurança do Paciente garantida e drástica redução de retrabalho das equipes de Farmácia e Engenharia Clínica.

🛠️ Tecnologias Utilizadas:
Oracle DatabaseSQL (PL/SQL)Tasy EMR

🚀 Otimização e Correção: Relatório de Consumo Diário por Local de Estoque (Tasy EMR)

1. O que é o projeto?

Reestruturação completa do relatório de Consumo Diário por Local de Estoque (WATE-1570). O objetivo é fornecer dados exatos para a parametrização de cotas e reposição inteligente de insumos.

2. O Problema Técnico

  • Dados Incorretos: Erro de Join (produto cartesiano) multiplicava os valores de consumo indevidamente.
  • Excesso de "Ruído": Listagem de itens com consumo zero gerava documentos de 74 páginas desnecessárias.
  • Filtros Ineficientes: Parâmetros de data e unidade hospitalar não funcionavam corretamente.

3. A Solução Aplicada

Refatoração do SQL corrigindo ligações entre tabelas, aplicação de cláusula HAVING SUM(qt_consumo) > 0, e uso da variável :nr_estab_v para filtragem dinâmica por unidade.

4. O Impacto na Ponta

Redução do relatório de 74 para 8 páginas úteis, precisão decisória para evitar falta ou excesso de medicamentos e agilidade operacional.

🛠️ Tecnologias e Ferramentas:
Tasy EMRSQL (Oracle PL/SQL)Gerenciador de Relatórios
Regras de Atributo • Tasy
